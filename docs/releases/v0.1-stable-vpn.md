# Liberty Shield v0.1 Stable VPN

**Tag:** `v0.1-stable-vpn`  
**Date:** 2026-05-02  
**Branch at tag:** `main` (`a3dff0e`)

---

## Core Engine

- **TCP relay** — per-session actor model; each session owns a `Channel<ByteArray>(256)` and a dedicated `queueJob` coroutine; PacketReader enqueues instantly without blocking
- **DNS cache** — in-memory LRU cache (256 entries, TTL 45 s); key strips transaction ID, response patched with incoming transaction ID on hit; thread-safe via `@Synchronized`
- **UDP relay** — one-shot fire-and-forget per packet; DNS timeout 800 ms, general UDP timeout 2000 ms
- **QUIC guard** — UDP 443 guard implemented but currently disabled; Chrome falls back to TCP naturally; persistent QUIC relay deferred
- **Crash guard** — per-packet `try/catch` in `PacketReader.run()`; `CancellationException` always rethrown; loop survives a single bad packet
- **Session cleanup** — `teardown()` idempotent via `CLOSED_FINAL` guard; `onClose()` removes session from `TcpSessionTable` and decrements `tcpSessionsActive`
- **Queue depth clamp** — `queueDepth.updateAndGet { maxOf(0, it - 1) }` prevents counter underflow on rapid teardown
- **Connect timeout** — reduced to 2500 ms; slow-connect warning logged at 500 ms and counted in `tcpSlowConnects`

---

## Performance Fixes

- **Removed heavy logging** — all hot-path `Log.d()` gated behind `VERBOSE_PACKET_LOGS = false`; `Log.w` / `Log.e` always active
- **Zero-copy TCP payload path** — `TcpPacketBuilder.buildData(buf, offset, len)` reads directly from the server read buffer without `copyOfRange()` allocation per chunk
- **UDP semaphore** — `Semaphore(32)` in `PacketForwarder` caps concurrent blocking `forwardUdp()` coroutines; excess packets dropped and counted in `udpConcurrencyDrops`
- **IO thread starvation fix** — root cause of "first load fast, subsequent loads slow": unbounded `scope.launch { forwardUdp() }` saturated `Dispatchers.IO` (64-thread cap), starving TCP `serverJob` coroutines; semaphore resolves this

---

## Stability Fixes

- **`NonCancellable` teardown in `TcpSession`** — `startServerReader()` wraps final `teardown()` in `withContext(NonCancellable)`; previously `CancellationException` was caught but the subsequent `sessionMutex.withLock` re-threw it, silently skipping teardown when `serverJob` was cancelled externally
- **VPN lifecycle decoupled from `ShieldService`** — removed `ACTION_STOP` from `ShieldService.onDestroy()`; battery management killing/restarting `ShieldService` no longer brings down the VPN relay
- **`PacketReader` auto-restart** — `ShieldVpnService` restarts `PacketReader` on crash with exponential backoff (500 ms base, 5 s max, 5 retries); `CancellationException` always propagated
- **Parser hardening** — `parseTcpSegment()` rejects corrupt `data-offset` field (`tcpHdrLen < 20`), clamps `payloadLen` to actual buffer size via `minOf(readU16(buf, 2), buf.size)`, bounds-checks `payloadOffset`

---

## Known Limitations

- **QUIC / HTTP3 not supported** — UDP 443 passes through the one-shot path; multi-packet QUIC flows are not relayed correctly; persistent UDP relay socket pool deferred
- **UDP persistent flows not implemented** — one-shot UDP only; each packet gets a new `DatagramSocket`; protocols requiring multi-packet UDP exchanges may not work reliably
- **No multi-hop routing** — VPN relay is single-hop direct; onion routing / Liberty Node integration not yet connected to the Android agent
- **No telemetry dashboard** — `VpnStats` metrics are logged every 5 s to logcat (`heartbeat`) but not surfaced in any UI or remote dashboard
