```bash
cargo build --release -p liberty-exit-node
cp target/release/liberty-exit-node /usr/local/bin/liberty-exit-node
```

Service file: `/etc/systemd/system/liberty-exit-node.service`

```ini
Description=Liberty Shield Exit Node
After=network-online.target

[Service]

EnvironmentFile=-/etc/liberty-exit-node/env

Restart=on-failure
RestartSec=5s
NoNewPrivileges=yes
ProtectSystem=strict
ProtectHome=yes
```

## Known Limitation: LIBERTY_PSK Not Configured

The server currently logs:

```
WARN liberty_exit_node: LIBERTY_PSK not set — running without authentication (skeleton mode only)
```

**Current PSK implementation status (accurate):**

| Location | Behavior |
|---|---|
| `config.rs:21` | Detects presence of `LIBERTY_PSK` env var only (`is_ok()`) — value is never read |
| `main.rs:18-19` | Logs a `WARN` if absent — no enforcement, server continues regardless |
| `metrics.rs` | `auth_failures` counter defined and exposed in health JSON — never incremented |
| `server.rs:22` | All Hello frames accepted with log "Hello frame received (no auth yet)" — zero auth logic |

**There is no PSK validation in the current codebase.** Any client can send a valid Hello frame
and it will be accepted. The `LIBERTY_PSK` variable is only checked for presence; its value is
never read, compared, or used in any computation.

This is acceptable for the v0.5.3 reachability test only. It must not be used with real traffic
or in any context where client identity matters.

## Next Recommended Milestone: v0.5.4

**Goal:** Remove the unauthenticated skeleton mode for real node operation.

Specific deliverables:
1. Read and validate `LIBERTY_PSK` value at startup (reject empty or short keys).
2. Implement HMAC-SHA256 authentication on Hello frames (token in payload or header extension).
3. Increment `auth_failures` counter on invalid or missing authentication.
4. Reject unauthenticated frames with a structured error log (no response sent to client).
5. Provision PSK via `/etc/liberty-shield/exit-node.env` (root-only, chmod 600).
6. Update systemd `EnvironmentFile` to reference that file.
7. Unit tests for: missing PSK, valid PSK hello, invalid PSK auth failure, counter behavior.
8. Update `send_hello` to accept `LIBERTY_HELLO_PSK` for authenticated test sends.

See `docs/runtime/v0.5.4-psk-auth-hardening-plan.md` for the full plan.
