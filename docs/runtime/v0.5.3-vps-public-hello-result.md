# Liberty Shield v0.5.3 — VPS Public UDP Hello Result

> **Status:** Engineering MVP. Not a production VPN.
> No Android tunnel. No traffic routing. No NAT forwarding.
> No encryption / authentication. No anonymity claim.

---

## 1. Purpose

First public-internet reachability test of `liberty-exit-node`.

```
Local PC ──UDP─→ VPS public IPv4:51820 ──→ liberty-exit-node
                                                   │
                                          "Hello frame received"
                                                   │
                                          /health: packets_rx++
```

What this test proves:
- VPS server binary starts and binds UDP/51820.
- A UDP Hello frame traverses the public internet from a local PC to the VPS.
- Server logs confirm the frame was received and parsed correctly.
- Health endpoint reflects updated metrics.
- `parse_errors` remains 0 for well-formed frames.

What this test does NOT prove:
- Real VPN routing (no packets forwarded to internet).
- IP masking (no Android client, no TUN).
- Encryption or authentication (frames are unencrypted).
- Production security.

## 4. VPS Setup Commands

Run these on the VPS over SSH. Do not paste credentials here.

### System update

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl build-essential pkg-config
```

### Install Rust

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
source "$HOME/.cargo/env"
rustc --version
```

### Clone / update repository

```bash
git clone <repo-url> liberty-shield
cd liberty-shield
git checkout feature/runtime-dashboard
git pull
```

If already cloned:

```bash
cd liberty-shield
git fetch origin
git checkout feature/runtime-dashboard
git pull
```

### Verify build on VPS

```bash
cargo check -p liberty-exit-node --bins
cargo test -p liberty-exit-node
```

Expected: 0 warnings, 13/13 tests pass.

### Run the server

```bash
LIBERTY_EXIT_BIND=0.0.0.0:51820 \
LIBERTY_EXIT_HEALTH_BIND=127.0.0.1:8081 \
LIBERTY_LOG_LEVEL=info \
cargo run -p liberty-exit-node
```

Expected startup log:

```
INFO liberty_exit_node: Liberty Exit Node starting bind=0.0.0.0:51820
INFO liberty_exit_node: health endpoint starting bind=127.0.0.1:8081
WARN liberty_exit_node: LIBERTY_PSK not set — running without authentication (skeleton mode only)
INFO liberty_exit_node: packet receive loop running
INFO liberty_exit_node::server: health endpoint listening health=127.0.0.1:8081
```

## 8. Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `curl /health` connection refused | SSH tunnel not open or server not running | Open SSH tunnel; verify server is running |
| `curl /health` times out | Server not listening or wrong tunnel port | Check `ss -ulnp` on VPS; check SSH tunnel binding |
| `send_hello` prints "sent OK" but no VPS log | Provider/OS firewall blocking UDP 51820 | Open UDP 51820 in cloud panel and UFW |
| `send_hello` fails ("send_to failed") | Wrong IP/port; local network issue | Verify `LIBERTY_HELLO_TARGET=IP:51820` |
| `parse_errors` increases | Frame mismatch or version mismatch | Ensure VPS runs same commit as local; check `git log` |
| "address already in use" | Another instance running | `pkill liberty-exit-node`; check `ss -ulnp` |
| `cargo build` fails on VPS | Rust toolchain missing or old | `rustup update`; re-run `source "$HOME/.cargo/env"` |
| Server starts but exits immediately | `run_health` bind failed; port in use | Check `ss -tlnp | grep 8081` |

## 9. Notes

- Health endpoint MUST remain on `127.0.0.1:8081` — never bind to `0.0.0.0`.
- Do not log PSK values or payload content.
- This test uses unencrypted frames; do not send sensitive data as payload.
- Run the server under `screen` or `tmux` on VPS so it persists after SSH disconnect.
