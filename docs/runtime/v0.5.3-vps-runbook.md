## Health Endpoint

Health is bound to `127.0.0.1:8081` on the VPS — never exposed publicly.
Access via SSH tunnel from local PC:

```bash
# Terminal A — open tunnel (keep running)
ssh -L 8081:127.0.0.1:8081 root@89.221.214.211

# Terminal B — query health
curl http://127.0.0.1:8081/health
```

Expected response:

```json
{
  "status": "ok",
  "packets_rx": 0,
  "packets_tx": 0,
  "bytes_rx": 0,
  "bytes_tx": 0,
  "active_sessions": 0,
  "parse_errors": 0,
  "auth_failures": 0
}
```

After sending Hello frames, `packets_rx` and `bytes_rx` will increment.
`parse_errors` must remain 0 for well-formed frames.
`auth_failures` is defined but currently always 0 (PSK auth not yet implemented).

## Update Binary on VPS

After a new commit on `feature/runtime-dashboard`:

```bash
cd /root/liberty-shield
git pull origin feature/runtime-dashboard
git log --oneline -3
cargo test -p liberty-exit-node              # must pass before installing
cargo build --release -p liberty-exit-node
systemctl stop liberty-exit-node
cp target/release/liberty-exit-node /usr/local/bin/liberty-exit-node
systemctl start liberty-exit-node
systemctl status liberty-exit-node --no-pager
```

## Troubleshooting

| Symptom | Check | Fix |
|---|---|---|
| Service not active | `systemctl status liberty-exit-node` | `systemctl start liberty-exit-node`; check journal |
| Health connection refused | SSH tunnel not open | Open tunnel: `ssh -L 8081:127.0.0.1:8081 root@89.221.214.211` |
| No log on send_hello | Firewall | `ufw status`; check `ss -ulnp \| grep 51820` |
| parse_errors increasing | Binary version mismatch | Pull latest, rebuild, restart service |
| Service exits on start | Port conflict / env error | `journalctl -u liberty-exit-node -n 20 --no-pager` |
| OOM / killed | Swap exhausted | `free -h`; increase swap if needed |

## Service File Reference

Located at: `/etc/systemd/system/liberty-exit-node.service`

```ini
[Unit]
Description=Liberty Shield Exit Node
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=root
EnvironmentFile=-/etc/liberty-exit-node/env
ExecStart=/usr/local/bin/liberty-exit-node
Restart=on-failure
RestartSec=5s
NoNewPrivileges=yes
ProtectSystem=strict
ProtectHome=yes
PrivateTmp=yes

[Install]
WantedBy=multi-user.target
```

`EnvironmentFile=-/etc/liberty-exit-node/env` — the `-` prefix means systemd continues if the
file is absent (skeleton mode). Once PSK auth is implemented this file will be required.

## Known Limitations (v0.5.3)

- No PSK authentication — all Hello frames accepted regardless of sender.
- No replay protection — duplicate frames processed without deduplication.
- No packet forwarding — Data frames logged and dropped.
- Metrics are in-memory only — reset to zero on service restart.
- `auth_failures` counter always remains 0 (PSK auth not implemented yet).
