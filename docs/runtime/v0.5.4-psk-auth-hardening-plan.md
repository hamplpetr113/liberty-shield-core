# Liberty Shield v0.5.4 — PSK Authentication Hardening Plan

## Goal

Remove the unauthenticated skeleton mode for real VPS operation.

After v0.5.4, the exit node must:
1. Reject all frames from clients that have not presented a valid PSK token.
2. Increment `auth_failures` on every rejected frame.
3. Log authentication errors without revealing key material.
4. Refuse to start in production mode without a valid PSK.

## Secret Provisioning

### Rule: PSK must never appear in source code, git history, or chat logs.

Provision via a root-only environment file on the VPS:

```bash
mkdir -p /etc/liberty-shield
chmod 700 /etc/liberty-shield

# Generate a 32-byte key (256 bits) — run on VPS, not locally
openssl rand -hex 32 > /etc/liberty-shield/exit-node.env.tmp
# Then edit to set the variable name:
echo "LIBERTY_PSK=$(cat /etc/liberty-shield/exit-node.env.tmp)" > /etc/liberty-shield/exit-node.env
rm /etc/liberty-shield/exit-node.env.tmp

chmod 600 /etc/liberty-shield/exit-node.env
chown root:root /etc/liberty-shield/exit-node.env
```

Contents of `/etc/liberty-shield/exit-node.env` (example format — do NOT commit):

```
LIBERTY_EXIT_BIND=0.0.0.0:51820
LIBERTY_EXIT_HEALTH_BIND=127.0.0.1:8081
LIBERTY_LOG_LEVEL=info
LIBERTY_PSK=<hex-encoded 32-byte key — generated on VPS>
```

### systemd integration

The service file already references this file:

```ini
EnvironmentFile=-/etc/liberty-exit-node/env
```

Update to require the file (remove `-` prefix once PSK is mandatory):

```ini
EnvironmentFile=/etc/liberty-shield/exit-node.env
```

After updating the service file:

```bash
systemctl daemon-reload
systemctl restart liberty-exit-node
```

## Required Code Changes

### `config.rs` — Read and validate PSK

```rust
pub struct Config {
    pub bind_addr: SocketAddr,
    pub health_bind: SocketAddr,
    pub psk: Option<[u8; 32]>,  // None = dev/local mode only
}

impl Config {
    pub fn from_env() -> Self {
        // ...
        let psk = match std::env::var("LIBERTY_PSK") {
            Ok(hex) => {
                let bytes = hex::decode(&hex)
                    .expect("LIBERTY_PSK must be valid hex");
                let arr: [u8; 32] = bytes.try_into()
                    .expect("LIBERTY_PSK must be exactly 32 bytes (64 hex chars)");
                Some(arr)
            }
            Err(_) => None,
        };
        // ...
    }
}
```

Requires adding `hex` crate to `Cargo.toml`.

### `main.rs` — Enforce PSK in non-dev mode

```rust
if cfg.psk.is_none() {
    if std::env::var("LIBERTY_DEV_MODE").is_ok() {
        warn!("DEV MODE: LIBERTY_PSK not set — authentication disabled");
    } else {
        error!("LIBERTY_PSK not set — refusing to start in production mode");
        error!("Set LIBERTY_DEV_MODE=1 to run without PSK for local testing only");
        std::process::exit(1);
    }
}
```

### `server.rs` — HMAC-SHA256 authentication on Hello frames

Authentication scheme for v0.5.4 (intentionally simple, to be replaced by Noise in v0.6):

Hello frame payload format with auth token:

```
| 32 bytes | N bytes |
| HMAC-SHA256 token | original payload |
```

Token = `HMAC-SHA256(key=PSK, msg=session_id || sequence || original_payload)`

Server-side validation:

```rust
fn validate_hello_auth(psk: &[u8; 32], frame: &Frame) -> bool {
    if frame.payload.len() < 32 {
        return false;
    }
    let (token, payload) = frame.payload.split_at(32);
    let expected = hmac_sha256(psk, frame.session_id, frame.sequence, payload);
    // constant-time comparison
    token == expected.as_bytes()
}
```

On failure:
```rust
METRICS.auth_failures.fetch_add(1, Ordering::Relaxed);
warn!(peer = %peer, session = frame.session_id, "Hello auth failure — frame rejected");
// do not send any response (no oracle)
```

### `send_hello.rs` — PSK-authenticated send

Add `LIBERTY_HELLO_PSK` env var support:

```bash
LIBERTY_HELLO_TARGET=89.221.214.211:51820 \
LIBERTY_HELLO_PSK=$(cat /etc/liberty-shield/exit-node.env | grep PSK | cut -d= -f2) \
cargo run -p liberty-exit-node --bin send_hello
```

If `LIBERTY_HELLO_PSK` is not set, send unauthenticated (for dev/testing only).

## Backward Compatibility Decision

| Mode | Behavior |
|---|---|
| `LIBERTY_PSK` set | Require valid HMAC token on all frames; reject without response |
| `LIBERTY_PSK` not set + `LIBERTY_DEV_MODE=1` | Warn and accept all frames (local dev only) |
| `LIBERTY_PSK` not set, no DEV_MODE | Error log + refuse to start |

**`LIBERTY_DEV_MODE=1` must never be set in the systemd service file on a real VPS.**

---

## Security Notes

1. PSK must be 32 bytes (256 bits) generated with a CSPRNG (`openssl rand -hex 32`).
2. PSK must never appear in source code, git commits, chat logs, or log output.
3. HMAC token comparison must use constant-time equality (`subtle::ConstantTimeEq`).
4. Server must not send any response on auth failure — no oracle for timing attacks.
5. Each frame's HMAC must cover `session_id || sequence || payload` to prevent replay
   of a valid token against a different sequence number or session.
6. The `EnvironmentFile` must be `chmod 600 root:root` on the VPS.
7. Noise XX handshake (v0.6) will supersede this PSK scheme entirely.
