# Liberty Shield v0.5.5 — Android Authenticated Hello

## Goal

Send the first authenticated Hello UDP frame from the Android app to the Liberty Shield
exit node on VPS ov9051 (89.221.214.211:51820), proving that Android can construct and
deliver a wire-compatible HMAC-SHA256 authenticated frame accepted by the Rust server.

This is the control-plane handshake stub. Not full VPN forwarding, not a real tunnel.

---

## Non-Goals

- Full packet forwarding through the exit node
- Noise XX handshake (v0.6)
- Real PSK management / key distribution
- UI for PSK configuration
- Multiple sessions or keepalive logic
- Per-frame auth for Data/Keepalive/Close (deferred to v0.6)

## Exact HMAC Canonical Message

Must match Rust `server/exit-node/src/auth.rs`:

```
session_id   (8 bytes, big-endian)
sequence     (8 bytes, big-endian)
msg_type     (1 byte = 0x01 for Hello)
original_payload (variable bytes)
```

Algorithm: HMAC-SHA256. Kotlin implementation uses `javax.crypto.Mac("HmacSHA256")`.

---

## Exact Frame / Payload Layout

Wire frame (big-endian), matches `server/exit-node/src/packet.rs`:

```
Offset  Len  Field
0       1    version = 0x01
1       1    msg_type = 0x01 (Hello)
2       2    flags = 0x0000
4       8    session_id (u64 big-endian)
12      8    sequence (u64 big-endian)
20      2    payload_len (u16 big-endian)
22      N    payload
```

Authenticated Hello payload:

```
[32-byte HMAC-SHA256 token][original_payload ("hello" = 5 bytes)]
```

Total authenticated Hello frame: 22 + 32 + 5 = **59 bytes**
(confirmed against VPS `bytes_rx` counter in v0.5.4 acceptance test)

## Local PSK Setup

1. Generate a 32-byte PSK on the VPS (where the real PSK lives):

   ```bash
   # VPS: print just the hex value of the existing PSK
   grep LIBERTY_PSK /etc/liberty-shield/exit-node.env | cut -d= -f2
   ```

2. Add to `android/local.properties` on your dev machine (this file is gitignored):

   ```
   LIBERTY_DEBUG_PSK_HEX=<paste 64 hex chars here>
   ```

3. Build the debug APK:

   ```bash
   cd android
   ./gradlew assembleDebug
   ```

4. Install and start VPN. `BuildConfig.DEBUG_PSK_HEX` will be embedded in the APK.

**Never commit `local.properties`. Never commit a real PSK in any source file.**

## How to Test Against VPS

### Before starting VPN on Android:

```bash
# VPS
curl http://127.0.0.1:8081/health
# Expected: packets_rx=0 (or previous baseline), auth_failures unchanged
```

### Start VPN on device. One authenticated Hello is sent automatically.

```bash
# VPS — within a few seconds of VPN start:
curl http://127.0.0.1:8081/health
# Expected:
#   packets_rx increased by 1
#   bytes_rx increased by 59
#   auth_failures unchanged
```

### Logcat filter on device:

```bash
adb logcat -s TunnelClient
# Expected line:
#   I TunnelClient: hello sent target=89.221.214.211:51820 frame_len=59 session=<ms> seq=1 auth=HMAC-SHA256
```

### Wrong PSK test:

Set an intentionally wrong 64-hex value in `local.properties`, rebuild, reinstall, start VPN.

```bash
# VPS:
curl http://127.0.0.1:8081/health
# Expected:
#   packets_rx unchanged
#   bytes_rx unchanged
#   auth_failures increased by 1
```

## Security Notes

1. `DEBUG_PSK_HEX` is embedded in the debug APK binary — anyone with APK access can extract it.
   Only use the real VPS PSK for internal dev testing. Rotate the PSK if the APK leaks.
2. `TunnelClient` never logs the PSK or HMAC token — only frame length, session ID, and sequence.
3. `HelloFrameBuilder.parsePsk()` rejects non-hex input and wrong length with `IllegalArgumentException`.
4. The `launchDebugHello()` call is wrapped in `runCatching` — a broken PSK or network error
   cannot crash or block VPN startup.
5. The VPN builder uses `.addDisallowedApplication(packageName)`, so `TunnelClient`'s UDP socket
   is routed outside the TUN (no loop). `protect()` is not required.
6. PSK authentication is symmetric / pre-shared. Replay protection and forward secrecy are
   deferred to the v0.6 Noise XX handshake which will replace this scheme entirely.

## Known Limitations

- `launchDebugHello()` is always triggered on VPN start if PSK is configured — there is no UI to
  disable it selectively. Acceptable for a dev stub; gate it behind a debug flag for beta.
- Session ID is derived from `System.currentTimeMillis()` — monotonic but not cryptographically
  random. For production, use a CSPRNG-generated u64.
- Only one Hello is sent per VPN start. Keepalive / periodic Hello not yet implemented (v0.5.7).
- If the network is not ready at VPN start time (unlikely given VPN establishment order), the
  send may fail silently. The next VPN start will retry.
