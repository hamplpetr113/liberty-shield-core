## Secret Provisioning

`LIBERTY_PSK` is stored only on the VPS at `/etc/liberty-shield/exit-node.env`.

**It must never be committed to git, logged, or included in chat output.**

The EnvironmentFile path referenced by the systemd service unit:

```
/etc/liberty-shield/exit-node.env
```

Permissions on VPS: `chmod 600`, `chown root:root`.

---

## Startup Enforcement

With `LIBERTY_PSK` set in the EnvironmentFile, the server logs:

```
auth: HMAC-SHA256 PSK
Liberty Exit Node starting bind=0.0.0.0:51820
packet receive loop running
```

Without `LIBERTY_PSK` and without `LIBERTY_ALLOW_UNAUTHENTICATED_DEV=1`, the server exits immediately:

```
ERROR LIBERTY_PSK is not set and LIBERTY_ALLOW_UNAUTHENTICATED_DEV is not set. ...
exit code: 1
```

`LIBERTY_ALLOW_UNAUTHENTICATED_DEV=1` is not set in the production service file.

## Auth Design (v0.5.4)

| Property | Value |
|---|---|
| Algorithm | HMAC-SHA256 |
| Key | `LIBERTY_PSK` — 32 bytes (256-bit), generated with `openssl rand -hex 32` on VPS |
| MAC covers | `session_id (8 BE) \|\| sequence (8 BE) \|\| msg_type (0x01) \|\| original_payload` |
| Hello payload layout | `[32-byte MAC token][original payload]` |
| MAC comparison | Constant-time via `hmac::Mac::verify_slice` |
| On auth failure | `auth_failures` incremented; no response sent (no oracle) |
| Non-Hello frames | Accepted unconditionally — per-frame auth deferred to v0.6 Noise XX |
