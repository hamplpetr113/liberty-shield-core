# Liberty Shield v0.5.2 — VPS Live Handshake Test Plan

> **Status:** Engineering MVP. Not a production VPN. No real traffic routing.
> No anonymity claim. No Android tunnel client.

---

## 1. Purpose

First live reachability test of `liberty-exit-node` on a real VPS.

What this test proves:
- The server binary starts and binds UDP/51820 on a VPS.
- A `Hello` frame sent from a local PC traverses the internet and is received.
- The server logs "Hello frame received" with the correct session ID.
- The health endpoint returns metrics confirming `packets_rx` incremented.
- Frame parse errors are counted and do not crash the server.

What this test does NOT prove:
- Real VPN routing (no packets forwarded to internet).
- IP masking (no Android client, no TUN).
- Encryption (frames are unencrypted at this stage).
- Production security of any kind.

---

## 2. Safety Boundaries

| Item | Rule |
|---|---|
| No payload forwarding | Server logs and drops all frames; no NAT forwarding yet |
| Health endpoint local only | Must remain bound to `127.0.0.1:8081`; never exposed publicly |
| No secrets in source | `LIBERTY_PSK` is provisioned via env, never hardcoded |
| No Android changes | `send_hello` is a standalone dev tool only |
| No anonymity claim | This is a frame reachability test, not a privacy guarantee |

---

## 3. Required VPS Information

Gather before starting. **Do NOT paste any of this into chat or source code.**

| Item | Where used |
|---|---|
| VPS public IPv4 | `LIBERTY_HELLO_TARGET` in send_hello |
| SSH username | local terminal only |
| SSH private key or password | local terminal only |
| Outbound interface name | nftables config (e.g. `eth0`, `ens3`) |
| UDP port (default 51820) | firewall rule + client config |

---

## 4. Secrets Warning

**NEVER share the following with anyone, including AI assistants:**

- SSH private key
- SSH password or sudo password
- VPS provider API token
- `LIBERTY_PSK` value (any key material)
- `/etc/liberty-exit-node/env` contents
- Any generated cryptographic material

These must not appear in source files, git history, chat logs, or terminal pastes.

## 5. Expected Test Result

### Server side (VPS terminal)

```
INFO liberty_exit_node: Liberty Exit Node starting bind=0.0.0.0:51820
INFO liberty_exit_node: health endpoint starting bind=127.0.0.1:8081
INFO liberty_exit_node: packet receive loop running
INFO liberty_exit_node: Hello frame received session=1
```

### Client side (local PC)

```
target:     <VPS_IP>:51820
frame_len:  27
session_id: 1
sequence:   1
msg_type:   Hello
sent OK
```

### Health endpoint (via SSH tunnel)

```json
{"status":"ok","packets_rx":1,"packets_tx":0,"bytes_rx":27,"bytes_tx":0,"active_sessions":0,"parse_errors":0,"auth_failures":0}
```

`packets_rx` must be ≥ 1 after each `send_hello` invocation.
`parse_errors` must remain 0 for well-formed frames.

---

## 6. Known Limitations at v0.5.2

- Frames are unencrypted; `LIBERTY_PSK` is not yet used for authentication.
- No replay protection; duplicate frames are processed.
- No session registry; `active_sessions` remains 0.
- No forwarding; all `Data` frames are logged and dropped.
- Health endpoint serves raw HTTP (no TLS); must stay localhost-only.
- Noise XX handshake is planned for v0.6.
